Why disk between every stage:
  • One failing source can't break the entire daily run.
  • Every artifact is auditable on GitHub (CSV diffs are human-readable).
  • You can re-run any single stage in seconds during development.

8Design Pattern Map

Where each pattern shows up in the codebase and what it earns us.

Pipeline / Pipes & Filters

Each stage in pipeline.py and auto_refresh.py reads files and writes files. Composable, observable, restartable.

Repository

Each external source has one module under src/data/ that returns a DataFrame, hiding API/scrape mechanics.

Strategy

baseline.py, advanced.py, farmer.py all share a train(features, horizon) → metrics contract.

Adapter

antalya_hal.py adapts a JS-rendered Vue page; hal_prices.py adapts an ASP form. Both produce the same DataFrame schema.

Composite

run_all_alerts orchestrates four rule strategies and merges their outputs into one severity-sorted list.

DTO (Data Transfer Object)

The Alert dataclass carries title, severity, impact, confidence, lead time as a strongly-typed structure.

Decorator (Cache)

@st.cache_data(ttl=300) on load_data() avoids reading every CSV on every Streamlit interaction.

Idempotent Writer

Every collector deduplicates by (date, source) on append. Re-running a refresh is always safe.

Soft Circuit-Breaker

try/except around every collector with status logged to refresh_log.csv. One failing source never breaks the day.
